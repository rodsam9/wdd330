Notes

Testing and Debugging

  • There will always be bugs and errors when it comes to progrmaming. Learning how to test and debug will help you be achieve tasks quicker.
  • There are different kinds of errors, such as system error(problem with the system), programmer error(maybe incorrect syntax or typos), and user error(user has entered data incorrectly)
  • Using exception can be very useful for a programmer to deal with errors.
  • You want the errors to fail loudly when programming, so it's important to ensure your code checks for errors or bugs.
  • Use strict mode to get better quality code. When not using it, it's like you're on sloppy mode coding.
  • A very basic form of debugging is using alert(), because a program is stopped till user clicks on OK.
  • Exception handling can be very useful to debug as well. When an exception occurs, the program ends with error message.

Single Page Applications

  • These are websites that re-render the content when clicking on links.
  • This is done without making a request to the server to fetch html.
  • Internal SPAs are limited to one entry, while location based SPAs you can share the link and not have to worry about the user not seeing the same view as you.

HTML5 Template

  • Template element is used to define a reusable fragment of HTML.
  • This holds on content that will be hidden when the page loads.
  • You can use JavaScript to display whatever is inside template tags.
