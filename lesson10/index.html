JS Execution Context

  • When Js code is being ran, it creates execution contexts.
  • There are two phases, the creating phase and the execution phase.

    • Creation Phase

  • During the creation phase, it creates a global object window in web browser, then creates a "this" object. It then sets up a memory for storing varialbes and references.

    • Execution Phase

  • JS engine runs the code line by line and assigns values to variables and runs function calls.
  • It creates a new function execution context for every function call. This is similar to global execution context, but this time it creates argument objects instead of a global object.

Call Stack

  • Call stacks help keep track of what function is being ran or called.

    • function announcement() {

    sayHelloWorld(); }

    function sayHelloWorld() {

    return "Hello World!"; }

    announcement();

In the example above, all the functions are ignored until announcement function is called. The announcement function gets added to the call stack.
